The Web Layer

How a URL becomes a web page — and how the app knows who you are

The Post Office

You type a URL. Somewhere, a Python function is about to run. But first — a letter has to find the right department.

📬
The Metaphor

Every web request is a letter arriving at the post office. Flask is the sorting room. It reads the address on the envelope (the URL path like /trades) and hands it to the right department. Each department prepares a response and sends it back. The letter never reaches the wrong department.

💡
Why this matters for you

When you want to add a new page, knowing the sorting room structure means you can tell AI "add a new route in the journal blueprint" — not just "add a page somewhere." Precision saves you a dozen back-and-forth messages.

"Who Are You?" — Two Auth Paths

Every protected route asks the same question: who is making this request? The answer depends on whether you are a browser or a script. Watch both conversations unfold.

Path 1: Browser
🌐
Browser

GET /trades

🛡️
@login_required

Show me your session cookie.

🌐
Browser

Here: session_id=abc123 (sent automatically with every request)

🛡️
@login_required

Decrypting... session has user_id=7. Looking up in database... that is Eric. You are in.

📋
trades.py

Loading Eric's trades now...

🛡️
Path 2: API Script
💻
API Script

GET /api/trades
X-API-Key: sk_xyz789abc

🛡️
@login_required

No session cookie. But I see an API key header. Let me hash it with SHA256...

🛡️
@login_required

Hash matches user_id=7 in users.api_key_hash. You are in.

📡
api.py

Returning JSON response with Eric's trades. No HTML — just clean data.

💻
API Script

Thanks. Processing the data now.

🛡️
💡
One decorator, two strategies

@login_required handles both paths. If a session cookie is present, it uses that. If an API key header is present, it hashes and looks it up. Admin routes additionally require @admin_required — which checks that users.is_admin == True in the database.
